## Practical Tips
- For every design, write requirements first, then scale, then data model, then APIs.
- Always mention trade-offs, bottlenecks, and failure handling.
- Compare alternatives instead of listing only one solution.
- Practice drawing the architecture from memory.
- Revisit weak topics with 1-page summaries and real interview questions.
